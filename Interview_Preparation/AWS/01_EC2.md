🖥️ AWS EC2 — Interview Questions & Answers
🌱 Basic Level

1. What is EC2?
EC2 (Elastic Compute Cloud) is a web service that provides virtual servers in the cloud. It allows you to launch and manage instances that can run applications, websites, or containers.

2. What is an AMI?
An AMI (Amazon Machine Image) is a pre-configured template that includes an operating system and software packages used to create EC2 instances.

3. What is the purpose of a key pair in EC2?
A key pair (public and private keys) is used for secure SSH access to your EC2 instance. The private key stays with you, and AWS stores the public key.

4. What is a Security Group?
A security group acts as a virtual firewall controlling inbound and outbound traffic to your EC2 instance.

5. What is an Elastic IP?
An Elastic IP is a static, public IPv4 address that stays the same even if you stop or restart your EC2 instance.

6. What is the difference between stopping and terminating an instance?
Stopping an instance shuts it down but retains the data on the EBS volume. Terminating deletes the instance and its storage (unless you’ve marked the EBS volume to persist).

7. What is EC2 user data?
User data is a script or configuration command that runs automatically when the instance is launched. It’s used for boot-time automation like installing packages or starting services.

8. What are the different pricing models for EC2?
On-Demand, Reserved, Spot Instances, and Savings Plans — each balances cost and flexibility differently.

**On-demand** is the most flexible but more expensive.

**Reserved** is cheaper if you know you’ll need the instance for a long time.

**Spot Instances** is the cheapest but can be interrupted.

**Savings Plans** offer flexibility with some discount.


9. What happens when you reboot an instance?
It restarts the OS but doesn’t affect data or configuration. Instance ID and Elastic IP remain unchanged.

10. What is an instance type in EC2?
An instance type defines hardware configuration — CPU, memory, and network performance (e.g., t2.micro, t3.medium, m5.large).

# What is a CIDR Block?

A CIDR block (Classless Inter-Domain Routing) is a way to represent a range of IP addresses. It allows you to specify a block of IP addresses in a more flexible and efficient way than traditional subnetting.

⚙️ Intermediate Level

1. What is the difference between EBS and Instance Store?
EBS is persistent block storage that remains even after stopping the instance. Instance Store is temporary; data is lost when the instance stops.

2. What are the types of EBS volumes?
gp3 (general purpose SSD), io1/io2 (high IOPS SSD), st1 (throughput optimized HDD), and sc1 (cold HDD).

3. How can you make EC2 instances highly available?
By placing them in multiple Availability Zones and using Load Balancers with Auto Scaling Groups.

4. How do you schedule EC2 start and stop automatically?
Use AWS Lambda with CloudWatch Events (cron jobs) to schedule start/stop times.

5. What is an EC2 placement group?
It’s a logical grouping of instances that influences placement — Cluster (low latency), Spread (high availability), or Partition (large-scale apps).

6. How can you connect Jenkins to an EC2 instance?
By using SSH keys or an IAM role, Jenkins can use EC2 as an agent or deploy directly to it.

7. How do you restrict SSH access to an instance?
By adjusting Security Group inbound rules to allow only specific IP addresses or CIDR ranges.

8. What is EC2 instance metadata?
It’s data about the instance, accessible at http://169.254.169.254/latest/meta-data/, such as instance ID, IP, or IAM role.

9. How can you back up an EC2 instance?
By creating an AMI image or taking snapshots of attached EBS volumes.

10. How can you share an AMI with another AWS account?
Modify AMI permissions to share with specific AWS accounts or make it public.

🚀 Advanced Level

1. How can you automate EC2 instance creation?
Using Infrastructure as Code tools like Terraform, CloudFormation, or Ansible to define and deploy EC2 configurations.

2. How do you secure access to AWS services from EC2 without storing credentials?
Attach an IAM Role to the EC2 instance — it provides temporary credentials automatically.

3. How do you monitor EC2 instances?
Use CloudWatch to monitor CPU, disk, memory, and network metrics. You can also set alarms for thresholds.

4. How do you recover from EC2 failure automatically?
Use Auto Scaling and Elastic Load Balancer; AWS replaces unhealthy instances automatically.

5. How do you perform Blue-Green deployment with EC2?
Maintain two environments (Blue = current, Green = new version) and switch traffic using the Load Balancer once the new version is tested.

6. How can EC2 communicate securely with RDS or S3?
Place them in the same VPC and use private subnets or VPC endpoints for private, secure communication.

7. What is the difference between user data and launch templates?
User data automates startup scripts; launch templates store instance configuration (AMI, SG, subnet, tags) for reuse.

8. How do you optimize EC2 cost?
Use Reserved or Spot instances, right-size instance types, and enable Auto Scaling.

9. How is EC2 used in CI/CD pipelines?
EC2 acts as a Jenkins agent to build, test, and deploy code or as a host for Docker/Kubernetes workloads.

10. What are some best practices for securing EC2?
Use IAM roles (not keys), restrict SSH access, keep OS patched, disable root login, and enable monitoring with CloudWatch and GuardDuty.
