🗂️ AWS S3 — Interview Questions & Answers
🌱 Basic Level

1. What is Amazon S3?
Amazon S3 (Simple Storage Service) is an object storage service that lets you store and retrieve any amount of data from anywhere over the internet.

2. What is an S3 bucket?
A bucket is a container where you store your objects (files). Every object in S3 must be stored inside a bucket.

3. What is an object in S3?
An object is a single file with metadata. It can be anything — text, images, logs, backups, etc.

4. How is data organized in S3?
Data is stored in a flat structure (not folders). “Folders” in S3 are just prefixes in the object key names.

5. What is the maximum size of a single S3 object?
Up to 5 TB. But uploads over 5 GB must use multipart upload.

6. What are the S3 storage classes?
Standard, Standard-IA (Infrequent Access), One Zone-IA, Glacier Instant Retrieval, Glacier Deep Archive, and Intelligent-Tiering.

7. What is versioning in S3?
Versioning keeps multiple versions of an object in the same bucket, allowing you to recover or roll back previous versions.

8. What is S3 durability and availability?
Durability = 99.999999999% (11 nines).
Availability = 99.99% for Standard class.

9. What is the AWS free tier for S3?
You get 5 GB of Standard storage, 20,000 GET requests, and 2,000 PUT requests per month free for 12 months.

10. How do you access S3 objects?
Through the AWS Console, CLI, SDKs, or REST API.

⚙️ Intermediate Level

1. How do you secure data in S3?
Use Bucket Policies, IAM permissions, and S3 Block Public Access settings to control access. You can also enable encryption.

2. What is S3 encryption?
Encryption can be at rest (SSE-S3, SSE-KMS, SSE-C) or in transit (HTTPS). SSE-KMS uses AWS Key Management Service for key control.

3. What is S3 Lifecycle Management?
It automatically transitions objects between storage classes or deletes them after a specific time.

4. What are S3 Access Logs?
They track requests made to your bucket — useful for auditing and analytics.

5. What is the difference between bucket policy and ACL?
Bucket policy = JSON-based rule at the bucket level.
ACL (Access Control List) = legacy access control for individual objects.

6. What is Cross-Origin Resource Sharing (CORS) in S3?
It defines how S3 content can be accessed from another domain (useful for web apps).

7. What is S3 Cross-Region Replication (CRR)?
It automatically copies objects from one bucket to another in a different AWS region for backup or latency reduction.

8. How do you host a static website using S3?
Enable “Static website hosting” in the bucket properties and make the bucket public. Then upload your HTML, CSS, JS files.

9. What is S3 Event Notification?
It triggers actions (like Lambda functions or SNS messages) when events occur, such as object uploads or deletions.

10. How can you restrict access to private content in S3?
Use pre-signed URLs that grant temporary access to objects.

🚀 Advanced Level

1. How do you use S3 in a CI/CD pipeline?
S3 can store build artifacts, Terraform state files, or host static frontends after Jenkins or CodePipeline deployment.

2. How do you ensure compliance or data governance with S3?
Use Object Locking, versioning, MFA Delete, and AWS Config for audit tracking.

3. What is S3 Object Lock?
It prevents deletion or modification of objects for a set period (Write Once Read Many – WORM compliance).

4. How can S3 integrate with CloudFront?
S3 serves as the origin for CloudFront, allowing faster global content delivery with caching.

5. How do you optimize cost in S3?
Use lifecycle policies, Intelligent-Tiering, and delete old versions. Monitor usage with AWS Cost Explorer.

6. How does S3 Transfer Acceleration work?
It speeds up uploads using Amazon’s edge network for long-distance data transfers.

7. How can you automate S3 backups?
Use AWS Backup service, Lambda scripts, or Lifecycle rules for periodic snapshots and replication.

8. What’s the difference between S3 Standard and Intelligent-Tiering?
Standard keeps data always accessible. Intelligent-Tiering automatically moves data to cheaper storage based on usage frequency.

9. How do you monitor S3 activity?
Use CloudWatch metrics, CloudTrail logs, and S3 Access Logs for auditing and alerts.

10. What are common S3 security best practices?

Block public access globally

Enable versioning and encryption

Use IAM roles over access keys

Turn on CloudTrail logging

Apply least privilege permissions
